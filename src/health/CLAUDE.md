# Health Module — 健康检查组件

**模块路径**: `src/health/`

## 概述

Health 模块提供组件级健康检查和状态快照功能，用于监控和诊断。

## 核心功能

### 1. ComponentHealth

单个组件的健康状态：

```rust
pub struct ComponentHealth {
    pub status: String,           // "ok" | "error" | "starting"
    pub updated_at: String,       // RFC3339 时间戳
    pub last_ok: Option<String>,  // 最后成功时间
    pub last_error: Option<String>, // 最后错误信息
    pub restart_count: u64,       // 重启计数
}
```

### 2. HealthSnapshot

全局健康快照：

```rust
pub struct HealthSnapshot {
    pub pid: u32,                        // 进程 ID
    pub updated_at: String,              // 更新时间
    pub uptime_seconds: u64,             // 运行时间
    pub components: BTreeMap<String, ComponentHealth>,
}
```

### 3. 操作函数

- `mark_component_ok(component)`: 标记组件正常
- `mark_component_error(component, error)`: 标记组件错误
- `bump_component_restart(component)`: 增加重启计数
- `snapshot()`: 获取当前快照
- `snapshot_json()`: 获取 JSON 格式快照

### 4. 状态转换

```
starting → ok
ok → error
error → ok
```

## 使用场景

- 健康检查端点
- 诊断信息输出
- 组件重启检测
- 运行时间统计
