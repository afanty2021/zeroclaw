# Service Module — 系统服务管理

**模块路径**: `src/service/`

## 概述

Service 模块提供跨平台的系统服务管理功能，支持安装、启动、停止和卸载 ZeroClaw 守护进程。

## 核心功能

### 1. InitSystem

支持的初始化系统：
```rust
pub enum InitSystem {
    Auto,     // 自动检测
    Systemd,  // systemd (user session)
    Openrc,   // OpenRC
}
```

### 2. 平台支持

- **macOS**: launchd 服务
- **Linux**: systemd (user) / OpenRC
- **Windows**: 计划任务

### 3. CLI 命令

- `service install`: 安装服务
- `service start`: 启动服务
- `service stop`: 停止服务
- `service restart`: 重启服务
- `service status`: 查看状态
- `service uninstall`: 卸载服务

### 4. macOS (launchd)

服务文件：`~/Library/LaunchAgents/com.zeroclaw.daemon.plist`

日志位置：
- `~/.zeroclaw/logs/daemon.stdout.log`
- `~/.zeroclaw/logs/daemon.stderr.log`

### 5. Linux (systemd)

服务文件：`~/.config/systemd/user/zeroclaw.service`

管理命令：
- `systemctl --user start zeroclaw.service`
- `systemctl --user enable zeroclaw.service`

### 6. Linux (OpenRC)

初始化脚本：`/etc/init.d/zeroclaw`

需要 root 权限安装：
- 创建 `zeroclaw` 系统用户
- 配置文件：`/etc/zeroclaw/config.toml`
- 工作目录：`/etc/zeroclaw/workspace`
- 日志目录：`/var/log/zeroclaw`

### 7. Windows

任务名称：`ZeroClaw Daemon`

包装脚本：`~/.zeroclaw/logs/zeroclaw-daemon.cmd`

## 安全设计

### OpenRC
- 系统用户隔离
- 文件权限控制 (0o750 / 0o755 / 0o600)
- 运行时写入权限验证

### systemd
- 用户会话隔离
- 自动重启配置

## 迁移支持

OpenRC 迁移：
- 从调用用户配置复制
- 自动检测现有配置

## 相关模块

- `src/daemon/`: 守护进程实现
- `src/config/`: 配置文件管理
