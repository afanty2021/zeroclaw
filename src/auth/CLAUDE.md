# Auth Module — OAuth 认证与配置文件管理

**模块路径**: `src/auth/`

## 概述

Auth 模块处理第三方服务的 OAuth 认证流程和 API 密钥管理，支持多配置文件、自动令牌刷新和加密存储。

## 核心功能

### 1. 支持的提供商

- **OpenAI Codex**: OAuth 2.0 + refresh token
- **Gemini/Google**: OAuth 2.0 + refresh token
- **Anthropic**: API token（静态）
- **通用提供商**: API token

### 2. 认证配置文件类型

```rust
pub enum AuthProfileKind {
    Token,  // 静态 API token
    OAuth,  // OAuth token + refresh token
}
```

### 3. AuthService

核心服务，提供：

- **存储令牌**: 支持新建和更新配置文件
- **获取令牌**: 自动处理过期和刷新
- **设置活跃配置**: 按提供商管理默认配置
- **删除配置**: 安全移除敏感数据
- **配置加密**: 可选的磁盘加密

### 4. OAuth 刷新机制

自动刷新逻辑：
1. 检查令牌是否将在 90 秒内过期
2. 获取分布式锁防止并发刷新
3. 重新加载配置以获取最新令牌
4. 执行刷新或使用退避策略
5. 更新配置文件并持久化

### 5. 退避策略

刷新失败时触发退避：
- 默认退避时间：10 秒
- 进程级退避状态跟踪
- 退避期间拒绝刷新请求

## 配置文件 ID 格式

```
<provider>:<profile_name>
```

示例：
- `openai-codex:default`
- `gemini:work`
- `anthropic:personal`

## 提供商规范化

别名映射：
- `codex` → `openai-codex`
- `claude` / `claude-code` → `anthropic`
- `google` / `vertex` → `gemini`

## 配置选择顺序

1. 显式覆盖参数
2. 提供商活跃配置
3. 提供商 `default` 配置
4. 提供商的第一个配置

## 存储位置

配置文件存储在：
```
<state_dir>/auth_profiles/
```

文件结构：
- `auth_profiles.json`: 加密的配置数据
- `.secret_key`: 加密密钥（如果启用加密）

## 安全特性

### 加密存储
- 可选的磁盘加密
- 密钥与配置分离存储
- 刷新时保持加密状态

### 敏感数据处理
- 日志中屏蔽令牌
- 调试信息中省略密钥
- JWT 解析用于提取账户 ID

## 相关模块

- `src/providers/`: 提供商实现
- `src/config/`: 配置加载
- `src/security/`: 密钥管理集成
