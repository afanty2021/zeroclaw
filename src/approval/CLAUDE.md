# Approval Module — 交互式审批工作流

**模块路径**: `src/approval/`

## 概述

Approval 模块实现了监督模式下的交互式审批工作流，在工具调用执行前请求用户许可，支持会话级别的"始终允许"白名单和审计日志。

## 核心功能

### 1. 审批决策类型

```rust
pub enum ApprovalResponse {
    Yes,    // 执行此调用
    No,     // 拒绝此调用
    Always, // 执行并添加到会话白名单
}
```

### 2. ApprovalManager

审批管理器，维护以下状态：

- **auto_approve**: 配置级自动批准列表
- **always_ask**: 配置级始终询问列表
- **autonomy_level**: 自治级别（Full/Supervised/ReadOnly）
- **session_allowlist**: 会话级白名单（由"Always"响应构建）
- **audit_log**: 所有决策的审计跟踪

### 3. 审批流程

1. **检查自治级别**:
   - `Full`: 永不提示
   - `ReadOnly`: 由其他模块拦截
   - `Supervised`: 进入审批流程

2. **检查 always_ask 列表**: 即使在会话白名单中也始终提示

3. **检查 auto_approve 列表**: 跳过提示

4. **检查会话白名单**: 之前的"Always"响应跳过提示

5. **默认行为**: 监督模式需要审批

## CLI 交互

CLI 通道支持交互式提示：

```
🔧 Agent wants to execute: shell
   command: rm -rf ./build/
   [Y]es / [N]o / [A]lways for shell:
```

- `Y` 或 `yes`: 执行此调用
- `A` 或 `always`: 执行并添加到会话白名单
- 其他输入: 拒绝

## 安全设计

### 审计日志

记录所有决策，包含：
- 时间戳
- 工具名称
- 参数摘要
- 决策类型
- 通道标识

### 参数摘要

敏感信息被截断以防止日志泄露：
- 字符串值截断至 80 字符
- 非对象值截断至 120 字符
- 超长内容用 `…` 标记

## 配置集成

与 `AutonomyConfig` 集成：

```toml
[autonomy]
level = "supervised"  # full | supervised | readonly
auto_approve = ["file_read", "memory_recall"]
always_ask = ["shell", "file_write"]
```

## 扩展点

当前仅 CLI 通道支持交互式提示。其他通道（Telegram、Discord 等）返回 `Yes` 默认响应。

## 关键约束

- `always_ask` 覆盖所有其他配置
- 会话白名单仅在当前会话有效
- 审计日志不可变
- 非 CLI 通道不支持交互式审批
