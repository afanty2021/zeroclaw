# RAG Module — 硬件数据表检索

**模块路径**: `src/rag/`

## 概述

RAG 模块实现硬件数据表的检索增强生成（RAG）管道，支持从 Markdown、文本和 PDF 文件中提取和检索硬件信息。

## 核心功能

### 1. HardwareRag

RAG 索引，管理：
- 数据表块
- 按板卡组织
- Pin 别名映射

### 2. DatasheetChunk

数据表块：
```rust
pub struct DatasheetChunk {
    pub board: Option<String>,  // 板卡标识
    pub source: String,         // 源文件路径
    pub content: String,        // 块内容
}
```

### 3. Pin 别名

Pin 别名映射：
```rust
pub type PinAliases = HashMap<String, u32>;
// "red_led" -> 13
```

### 4. 加载数据表

```rust
pub fn load(workspace_dir: &Path, datasheet_dir: &str) -> Result<Self>
```

支持的格式：
- Markdown (`.md`)
- 文本 (`.txt`)
- PDF (`.pdf`, 需要 `rag-pdf` 功能)

### 5. Pin 别名解析

从 Markdown 解析：
- `## Pin Aliases` 部分
- `| alias | pin |` 表格
- `alias: pin` 键值对

### 6. 检索功能

```rust
pub fn retrieve(&self, query: &str, boards: &[String], limit: usize) -> Vec<&DatasheetChunk>
```

使用关键词匹配和板卡过滤。

### 7. Pin 别名上下文

```rust
pub fn pin_alias_context(&self, query: &str, boards: &[String]) -> String
```

为查询构建 Pin 别名上下文。

## 功能门控

- `rag-pdf`: 启用 PDF 支持

## 使用场景

- 硬件开发辅助
- Pin 配置查询
- 外设接口查找

## 相关模块

- `src/hardware/`: 硬件发现
- `src/peripherals/`: 外设控制
- `src/memory/chunker/`: 文本分块
